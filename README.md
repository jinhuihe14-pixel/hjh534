# AI智能用户召回系统

基于多维度用户行为数据，搭建**用户画像、流失预测、智能分群、个性化召回**一体化AI系统，融合分类模型、聚类算法、推荐算法，实现千人千面的智能运营。

## 核心功能

### 1. 用户画像系统
- 多维度特征：基础属性、行为特征、消费特征、内容偏好
- 数据脱敏：用户ID哈希化，敏感信息加密存储
- 实时更新：支持增量特征计算

### 2. 流失预测模型
- **模型算法**：Gradient Boosting + 时序特征
- **预测周期**：7天、15天、30天流失概率
- **风险等级**：低风险 / 中风险 / 高风险 / 极高风险
- **流失归因**：内容不合预期 / 互动减少 / 活跃度下降 / 付费意愿低
- **模型效果**：AUC > 0.95，F1 > 0.9

### 3. 用户分群聚类
- **算法**：K-Means 无监督聚类
- **群体数量**：6个核心用户群体
- **群体画像**：高价值活跃用户、潜力付费用户、普通活跃用户、轻度沉睡用户、深度流失用户、新用户
- **特征画像**：自动提取每个群体的Top核心特征

### 4. 个性化召回策略
- **差异化策略**：针对不同用户群体配置不同召回方案
- **智能文案**：基于流失原因和内容偏好生成个性化推送文案
- **优惠券配置**：VIP折扣 / 现金券 / 积分奖励 / 专属内容
- **最佳触达**：基于活跃时段预测最佳推送时间
- **优先级排序**：综合价值、风险等级、召回潜力计算优先级

### 5. A/B实验框架
- **多组对照**：支持多版本召回策略并行实验
- **实时统计**：曝光量、转化量、转化率
- **用户分流**：按流量权重自动分配实验组
- **指标上报**：支持自定义指标埋点

## 快速开始

### 1. 安装依赖

```bash
pip install -r requirements.txt
```

### 2. 初始化系统（生成数据 + 训练模型）

```bash
python init_system.py
```

这将：
- 生成 2000 个模拟用户，60 天行为数据
- 构建 19 维用户特征
- 训练流失预测模型（GBDT）
- 训练用户聚类模型（K-Means, K=6）
- 初始化 2 个默认 A/B 实验

### 3. 启动服务

```bash
python run.py
```

### 4. 访问运营后台

打开浏览器访问：http://localhost:8000

## API 接口

### 系统管理
- `POST /api/system/init` - 初始化系统
- `GET /api/system/status` - 获取系统状态

### 用户管理
- `GET /api/users` - 获取用户列表（分页、筛选）
- `GET /api/users/{user_id}` - 获取用户详细画像
- `GET /api/users/{user_id}/recall` - 获取用户召回方案

### 统计分析
- `GET /api/statistics/overview` - 数据概览
- `GET /api/statistics/churn` - 流失统计分析

### 用户分群
- `GET /api/clusters` - 获取所有用户群体及画像

### 召回策略
- `POST /api/recall/batch` - 批量生成召回方案

### A/B实验
- `GET /api/ab/experiments` - 获取实验列表
- `POST /api/ab/experiments/{id}/start` - 开始实验
- `POST /api/ab/experiments/{id}/stop` - 停止实验
- `GET /api/ab/experiments/{id}/results` - 获取实验结果
- `POST /api/ab/experiments/{id}/assign/{user_id}` - 分配用户到实验组
- `POST /api/ab/experiments/{id}/metric/{variant}/{metric}` - 上报实验指标

## 技术栈

- **后端框架**：FastAPI + Pydantic
- **机器学习**：scikit-learn (GBDT, K-Means)
- **数据处理**：pandas + numpy
- **前端**：Vue 3 + ECharts
